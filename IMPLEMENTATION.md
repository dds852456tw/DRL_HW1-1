# HW1-1: 網格地圖開發 (Grid Map Development) 實作紀錄

## 需求總覽
1. 開發一個大小為 n x n 的網格地圖，透過 Flask 網頁應用程式顯示。
2. 允許用戶輸入並指定維度 n (範圍從 5 到 9)。
3. 使用者介面互動：
   - 第一次點擊：設定為「起始單元格」(顯示為綠色)。
   - 第二次點擊：設定為「結束單元格」(顯示為紅色)。
   - 後續點擊：設定為「障礙物」(顯示為灰色)，最多可設置 n - 2 個。
   - 點擊已選取的格子可取消選取並恢復為白色。

## 開發過程
### 1. 專案建立與套件安裝
- 建立專案資料夾並初始化 Git。
- 安裝 Flask 框架 (`pip install flask`) 用於建構輕量級後端伺服器。

### 2. 後端開發 (Flask)
- 建立 `app.py`，負責提供路由 (Routing)。
- 設定根目錄路由 `/` 回傳前端頁面 `index.html`。
- 啟用開發模式並綁定 `0.0.0.0:5000` 以利本地端測試。

### 3. 前端開發 (HTML/CSS/JavaScript)
- **版面設計**：在 `templates/index.html` 中實作基本版面，包含一個輸入維度的工作列 (帶有 n 範圍輸入框與產生按鈕)。
- **網格渲染**：
  - 透過 JavaScript 動態生成網格元素 (`div`)，並利用 Flexbox 或 CSS Grid 來維持正方形比例與排列。
  - 為每個格子標上 1 到 $n^2$ 的數字 ID。
- **互動邏輯 (JavaScript)**：
  - 核心變數追蹤：`startCell` (起點), `endCell` (終點), `obstacles` (障礙物陣列), `maxObstacles` (最大障礙物數量)。
  - 點擊事件：依據依序的條件判斷目前未設定的狀態(起點 -> 終點 -> 障礙物)，並對點選的節點加上對應的 CSS Class 產生顏色變化 (綠色、紅色、灰色)。
  - 取消邏輯：判斷該節點是否已含有狀態 Class，若有則移除，使流程可逆。

### 4. 測試與部署
- 執行本地伺服器測試點選邏輯，確認起、終、障礙物(最多 n-2 個)的功能如預期運行。
- 透過瀏覽器代理工具驗證畫面行為後，提交並推送 (Push) 至 GitHub repository。
